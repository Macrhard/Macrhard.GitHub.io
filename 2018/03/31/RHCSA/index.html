不同的系统常用的打包压缩方式是不同的
•Windows 常用rar
•Mac　常用　zip
•Linux 常用　tar.gz

tar 是linux中最常用打包工具，压缩是gzip软件
f 必须放在最后其他参数的顺序随意

gzip
• tar与gzip命令结合可以实现文件的打包和压缩
• tar 只负责打包，gzip只负责压缩
• 用gzip压缩tar包后，其扩展名一般是xxx.tar.gz这是linux中最常见的压缩文件格式

tar中有一个 -z选项可以调用gzip，从而方便实现打包压缩和解压缩的功能

RHEL 7在单用户模式下重置root密码

  1. 按e进入grub界面，在grub界面中修改linux16这一行的参数，将ro及其之后的这段内容删除，添加 rw rd.break init=/sysroot/bin/sh这段参数。修改完后按Ctrl+x键启动，让系统进入单用户启动模式。
  2. 系统会出现switch_root:/#界面的命令行提示符，输入chroot /sysroot更换根
  3. 输入passwd root更改root密码，或者以显示形式更改密码，输入echo 123456 |passwd --stdin root。
  4. 修改完root密码后，让SElinux系统打标签，记住密码的改动。输入touch /.autorelabel。
  5. 连续输入两次exit让SELinux自动把服务全部打开，重新打标签。开始打标签时会有百分制的进度显示。
  6. 打标签完成后系统会重新启动，启动后用root用户登录，输入更改后的密码即可。

存储结构与磁盘划分

Linux系统中常见的文件系统有：Ext3 Ext4 XFS

在Linux系统中，目录、字符设备、块设备、套接字、光驱、打印机等都被抽象成了文件，“Linux系统中一些都是文件”。

常见硬件设备及其文件名称

硬件设备 文件名称
IDE设备 /dev/hd[a-d]
SCSI/SATA/U盘 /dev/sd[a-p]
打印机 /dev/lp[0-15]
光驱 /dev/cdrom[1-15]
鼠标 /dev/mouse

一块全新的硬盘需要先分区，再格式化才能挂载

挂载是使用硬件设备的最后一步。

  1. mount

    用挂载文件系统，能够把硬盘设备或分区与一个目录文件进行关联，然后就能在该目录中对设备进行存取操作了。

    mount 文件系统 挂载目录

    -a 挂载所有在/etc/fstab中定义的文件系统

    -t 指定文件系统的类型（较新的Linux系统会自动进行判断，一般不需要此参数）

    配置/etc/fstab文件，开机自动挂载文件系统设备。

    | 字段 | 意义 |
    | ———— | —————————————————————————————— |
    | 设备文件 | 一般为设备的路径+设备名称，也可以写唯一识别码UUID |
    | 挂载目录 | 指定挂载到的目录，需要挂载前创建好 |
    | 格式类型 | 制定文件系统的格式，如ext3、ext4、xfs、swap、iso9660(此为光盘设备) |
    | 权限选项 | 若设置defaults，则默认权限为：rw，suid，dev，exec，auto，nouser，async |
    | 自检 | 如为1则开机后对磁盘进行自检，为0不自检 |
    | 优先级 | 若自检字段设为1则可对多块硬盘设置优先级，不设则为0 |

如配置本地镜像yum库对/etc/fstab文件进行配置，设置自动挂载镜像到/mnt

配置自动挂载

vim /etc/fstab

/dev/cdrom /mnt iso9660 defaults,loop 0 0
